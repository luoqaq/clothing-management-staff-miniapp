# AGENTS 工作规范

## 作用范围
- 本文件对 `clothing-management-staff-miniapp` 仓库生效。

## 环境基线
- 本仓库默认使用 `npm` 管理依赖和脚本。
- 小程序技术栈为 `Taro + React + TypeScript`。
- 后端默认接入 `TARO_APP_API_BASE_URL=http://127.0.0.1:3000/api`。

## 常用命令
- 安装依赖：`npm install`
- 微信小程序开发构建：`npm run dev:weapp`
- 微信小程序生产构建：`npm run build:weapp`

## 必要会话流程
1. 每次新会话开始时，先读取 `PROJECT_MEMORY.md`。
2. 同时检查 `README.md`；如果本次任务涉及项目启动、构建、部署、目录结构、接口约定或标准工作流，优先读取 `README.md`。
3. 仅当本次会话产生了值得复用的高价值信息时，才在会话结束前更新 `PROJECT_MEMORY.md`。
4. 记忆更新采用“会话结束统一总结后再写入”的方式，不在处理中频繁追加零散记录。
5. 收尾回复固定包含三项：
   - 改了什么
   - 如何验证
   - 剩余风险或待办

## 变更边界
- 仅修改与当前需求直接相关的文件。
- 未经用户要求，不做顺手重构。
- 如果发现与当前任务冲突的意外本地改动，先暂停并确认。

## 使用方式提醒
- 如果发现用户当前使用方式低效、容易误操作，或与仓库现有工作流不一致，应直接指出。
- 给出更优用法时，优先提供当前项目可直接执行的命令、路径或调用方式，不只给抽象建议。
- 如果用户的做法能完成任务但成本更高，也应主动说明更省事、更稳定或更可复用的替代方案。
- 涉及联调、构建、环境配置或上传链路排障时，优先提醒已有标准入口和后端约定，避免本地假设漂移。

## 验证规则
- 代码修改后，至少执行一项相关验证。
- 若依赖未安装或构建环境缺失，必须明确说明阻塞点。

## 安全规则
- 回复中不得输出密钥、密码、令牌等敏感信息。
- 未经明确要求，不修改环境配置文件。

## Git 规则
- 提交信息清晰可读，优先使用 Conventional Commit 风格。
- 未经明确要求，不执行 `commit --amend`。
- 未经明确要求，不执行强推（force push）。

## 记忆更新规则
- 内容保持简洁、事实化。
- 仅记录高价值信息：环境或配置变化、运行与验证结果、问题与根因、关键决策和约束。
- 只有当改动对后续会话、联调、部署、排障或方案延续有明显帮助时，才更新记忆。
- 记忆更新时机固定为：本次会话工作基本结束后，统一总结并写入。
- 更新 `PROJECT_MEMORY.md` 时使用固定模板：会话日期、变更内容、验证结果、遗留问题或风险。
- 避免重复陈旧或低价值信息。

## 项目特有约束
- 本期小程序定位为 ToB 店铺运营协同端，不引入 ToC 用户模型。
- 复用现有服装管理系统账号、商品、库存、订单与 COS 图片上传链路。
- 图片上传默认先在小程序端压缩，再走后端签发凭证直传 COS。
